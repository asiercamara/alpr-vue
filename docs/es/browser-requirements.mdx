ALPR Vue ejecuta modelos de IA directamente en tu navegador usando **WebAssembly** — una tecnología que permite que el código compilado se ejecute a velocidad casi nativa sin necesidad de plugins. También usa **OffscreenCanvas** para procesar fotogramas de vídeo en un hilo en segundo plano, manteniendo la interfaz fluida mientras se ejecuta la detección. Ambas funcionalidades deben estar disponibles en tu navegador para que la aplicación funcione.

## Compatibilidad con navegadores

| Navegador | Escritorio | Móvil |
|---|---|---|
| Chrome | ✅ Compatible | ✅ Compatible |
| Edge | ✅ Compatible | ✅ Compatible |
| Firefox | ✅ Compatible | ✅ Compatible |
| Safari 16+ | ✅ Compatible | ✅ Compatible |
| Safari (versiones anteriores a 16) | ⚠️ Parcial | ⚠️ Parcial |

<Warning>
  Los navegadores que no admiten **WebAssembly** u **OffscreenCanvas** no podrán ejecutar ALPR Vue. Esto incluye Internet Explorer y versiones antiguas de Safari. Si la aplicación no carga o los modelos nunca se inicializan, comprueba que tu navegador esté actualizado.
</Warning>

## Funcionalidades requeridas del navegador

ALPR Vue depende de tres capacidades del navegador:

- **WebAssembly** — ejecuta los modelos de detección de matrículas YOLO y de OCR localmente, dentro de tu navegador
- **OffscreenCanvas** — procesa fotogramas de vídeo fuera del hilo principal para que la interfaz permanezca receptiva
- **Contexto seguro (HTTPS o localhost)** — requerido por los navegadores antes de conceder acceso a la cámara

<Note>
  El acceso a la cámara requiere un **contexto seguro** — la página debe servirse mediante HTTPS o desde localhost. Si abres ALPR Vue mediante una conexión HTTP simple, el botón de cámara no funcionará. La carga de imágenes y vídeos sí funciona sin contexto seguro.
</Note>

## Especificaciones recomendadas del dispositivo

ALPR Vue funciona en una amplia gama de dispositivos, pero el rendimiento depende de la capacidad de procesamiento:

- **Smartphone**: Cualquier dispositivo Android o iOS moderno (lanzado aproximadamente en los últimos cuatro años) con aceleración de hardware activada en el navegador funciona bien
- **Ordenador de escritorio o portátil**: Cualquier ordenador moderno gestiona los modelos cómodamente; la aceleración gráfica por hardware ayuda a mantener un procesamiento de fotogramas fluido
- **Aceleración de hardware**: Activa la aceleración por GPU en la configuración de tu navegador si está disponible — esto mejora significativamente la velocidad de inferencia del modelo tanto en escritorio como en móvil

## Requisitos de Node.js (solo para alojamiento propio)

Si estás compilando o alojando ALPR Vue tú mismo, tu entorno necesita:

- **Node.js** `^20.19.0` o `>=22.12.0`
- **pnpm** como gestor de paquetes

Estos requisitos aplican solo al entorno de compilación, no a los usuarios que acceden a la aplicación ya desplegada.
